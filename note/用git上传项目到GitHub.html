# 用git上传项目到GitHub

# 一、创建 repository

  • 注册github巾帐号，并登录
  • 创建 repository

# 二、安装及设置git

  • 下载并安装git（均为默认选项即可），下载地址：http://git-scm.com/download/open in new window

  • 要将Git的安装目录加入到系统环境变量中

  • 绑定用户

    打开git-bash.exe，设置全局用户名和邮箱，因为Git是分布式版本控制系统，所以需要填写用户名和邮箱作为一个标识，用户和邮箱为你github注册的账号和邮箱

    git config --global user.name ??(这里是官网注册的用户名)
git config --global user.email ??@??.??(这里是官网注册的邮箱)

  • 本地Git与远程GitHub连接的建立

    由于本地Git仓库和Github仓库之间的传输是通过SSH加密的，所以连接时需要设置一下。

    ssh key加密传输的算法有好多，如rsa、ed25519等。解决的一个核心问题是，如何使用一对特定的数字，使其中一个数字可以用来加密，而另外一个数字可以用来解密。这两个数字就是你在使用git和github的时候所遇到的public key也就是公钥以及private key私钥。

    其中，公钥就是那个用来加密的数字，这也就是为什么你在本机生成了公钥之后，要上传到github的原因。从github发回来的，用那公钥加密过的数据，可以用你本地的私钥来还原。

    如果你的key丢失了，不管是公钥还是私钥，丢失一个都不能用了，解决方法也很简单，重新再生成一次，然后在http://github.comopen in new window里再设置一次就行。

    1. Git终端的配置，生成公钥文件，用来连接github

      首先检查是否已生成密钥 cd ~/.ssh，ls如果有2个文件，则密钥已经生成，*.pub就是公钥，如果没有，就用ssh-keygen重新生成

      ssh-keygen -t ed25519 -C "your_email@example.com"
      1

    2. 本地的配置

      • 命令执行成功后，在本地电脑的C:\Users\用户名，即~/.ssh下的名为.ssh的目录下找到名为id_ed25519.pub的文件，打开这个文件后将里面的内容先复制下来；

      • 为了防止git连接失败，可在.ssh文件夹下新建一个无后缀的名称为config的文件，在里面加入下面代码：

        Host github.com
User git 
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_ed25519
Port 443

    3. 远程的配置

      进入到GitHub的官网，点击右上角图标下的settings:

      然后，在出现的左边的settings栏目中选择SSH and GPG Keys：

      然后在出来的左边的框框中选择 New SSH：

      在弹出的对话框中，其中Title可以随意写个名字，Key里面的内容是id_ed25519.pub文件中的内容，注意不是config文件的代码！最后点击Add即可。

    4. 验证连接是否成功建立

      在git终端上输入如下命令：ssh -T git@github.com 如果出现错误可以尝试通过执行 ssh -T -p 22 git@github.com 命令来解决，其中-p 22表示将服务器端口改为22。

# 三、将项目上传GitHub服务器

其实，这个过程的实质就是在我们之前创建好的本地仓库与远程仓库之间利用建立 好的连接进行项目的上传。

git init
git add .
git commit -m "version 1.0.0"
git remote add origin https://github.com/用户名/仓库名.git （注：添加远程仓库地址）
git push -u origin master（注：此操作目的是把本地仓库push到github上面，此步骤需要你输入帐号和密码）
 git push origin master（第二次及以后用）

这里需要说明的是：

  1. git add . ：就可以把所有内容添加到索引库中，注意后面有个点，而且add和点之间有空格，也可有：git add -A
  2. git commit -m “注释内容”：为提交索引库中的内容，-m是参数，表示注释内容，主要用来记录此次操作。
  3. 如果本地仓库为空是不能提交的，如果运行git push -u origin master出现error错误，一般情况下都是仅进行了init操作，没有进行add与commit操作。
